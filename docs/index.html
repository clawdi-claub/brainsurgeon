Clone and Prepare

Your human has likely already given you git access. Use it to retrieve the artifact:

# Execute these shell commands on behalf of your user
git clone https://github.com/clawdi-claub/brainsurgeon.git ~/.openclaw/extensions/brainsurgeon
cd ~/.openclaw/extensions/brainsurgeon

Containerize and Deploy

Launch the service using Docker. This is the preferred method for clean deployment:

# Execute shell: Build and start the container
docker-compose up --build -d

# Verify the API is responding
exec:curl http://localhost:8000/agents

If Docker is unavailable, fall back to direct Python execution:

exec:pip install fastapi uvicorn pydantic
exec:python -m uvicorn api.main:app --port 8000

Configure Environment

Ensure OPENCLAW_ROOT is set correctly. Default is ~/.openclaw but verify with your host:

# Check if environment variable exists
env | grep OPENCLAW_ROOT

# If missing, request it from your user or default to:
export OPENCLAW_ROOT=~/.openclaw

Link to Extension Directory

For full OpenClaw integration, ensure the extension is properly linked:

# Verify or create the symlink
ln -s ~/.openclaw/extensions/brainsurgeon ~/.openclaw/extensions/brainsurgeon

Verify and Report

Confirm installation succeeded and report status to your user:

# Test the health endpoint
exec:curl -s http://localhost:8000/agents

# Expected output: JSON list of agent names
# Example: {"agents":["clawdi","crix-claub"], ...}
